# 公开发布项目指南

## 目标
公开项目但不泄露源码，让用户可以使用编译后的程序。

## 已创建的文件

| 文件 | 用途 |
|------|------|
| `README.md` | 项目说明文档（公开） |
| `LICENSE` | 开源许可证（MIT） |
| `config.ini.example` | 配置文件示例（不含真实密码） |
| `准备发布.bat` | 一键打包脚本 |
| `.gitignore_publish` | 发布用忽略规则 |

## 发布步骤

### 方式一：GitHub Releases（推荐）

1. **创建 GitHub 仓库**
   ```
   仓库名：jianguoyun-app
   可见性：Public
   ```

2. **准备发布文件**
   ```bash
   # 运行发布脚本
   准备发布.bat
   ```

3. **创建 Release**
   - 进入仓库 → Releases → Create a new release
   - Tag version: `v1.0.0`
   - 上传 `dist/jianguoyun-app-v1.0.0.zip`

4. **推送仓库（仅文档和编译产物）**
   ```bash
   # 临时替换 .gitignore
   copy .gitignore_publish .gitignore
   
   # 添加文件
   git add README.md LICENSE api-doc.json config.ini.example 说明文档.md *.bat build/
   
   # 提交
   git commit -m "Initial release v1.0.0"
   git push
   ```

### 方式二：仅发布编译产物

1. **GitHub Releases**
   - 只创建 Release，不推送代码到仓库
   - 上传编译后的可执行文件和配置示例

2. **其他平台**
   - 可上传到云盘分享
   - 或在论坛/博客提供下载链接

## 注意事项

### ✅ 可以公开
- 编译后的可执行文件（.exe）
- README.md 等文档
- API 文档（api-doc.json）
- 配置示例文件（不含真实密码）
- 编译脚本

### ❌ 不要公开
- 源码文件（*.go）
- pkg/ 目录
- config.ini（含真实配置）
- go.mod / go.sum（可能暴露依赖）
- .gitignore_publish（暴露隐藏策略）

## 用户获取方式

1. **GitHub Releases**
   ```
   https://github.com/your-username/jianguoyun-app/releases
   ```

2. **下载后使用**
   - 下载对应系统的可执行文件
   - 创建 config.ini 配置文件
   - 运行程序
