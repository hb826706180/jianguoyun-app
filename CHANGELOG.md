## 版本说明

### 版本号规则
遵循语义化版本规范 (Semantic Versioning): `主版本号。次版本号。修订号`

- **主版本号**: 不兼容的 API 变更
- **次版本号**: 向后兼容的功能新增
- **修订号**: 向后兼容的问题修正

### 部署方式
使用自动化部署脚本 `推送.ps1`:
```powershell
./推送.ps1
```

脚本会自动完成以下操作:
1. 检查当前 Git 状态
2. 添加更改的文件
3. 提交更改
4. 推送到 GitHub
5. 自动生成版本号
6. 创建 Git 标签
7. 推送标签
8. 创建 GitHub Release

### 注意事项
- 需要设置 `GITHUB_TOKEN` 环境变量才能自动创建 Release
- 编译产物会自动包含在 Release 中
- 敏感配置文件 (如 `config.ini`) 不会被推送
