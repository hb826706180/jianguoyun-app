# 代码维护通用规范（功能删除）

## 一、核心清理流程

1. 代码清理：删除功能相关的处理器、服务、控制器、模型等核心代码，移除对应路由/接口、配置项及第三方依赖；
2. 文档更新：删除README、API文档（如Swagger）、使用示例中功能相关内容，清理代码注释/文档字符串；
3. 测试清理：删除功能对应的单元/集成测试代码，清理测试数据和测试配置；
4. 构建部署：更新构建脚本、部署配置、CI/CD流程，移除功能相关步骤。

## 二、快速检查清单

### 代码文件

- 处理器/控制器、服务/逻辑、数据模型文件
- 路由定义、配置文件、辅助工具类

### 文档文件

- 项目README、API文档、技术/使用指南

### 测试文件

- 单元/集成测试文件、测试数据/配置

### 构建部署

- 构建脚本、CI/CD配置、Docker/环境配置

## 三、最佳实践

1. 模块化设计：功能代码独立封装，命名统一，便于识别和清理；
2. 文档同步：删除功能时同步更新所有相关文档，避免文档与代码不一致；
3. 依赖检查：用静态分析工具排查依赖，评估删除操作的影响范围；
4. 测试验证：删除后通过自动化测试确认无残留问题。

## 四、注意事项

1. 操作前备份代码库，通过版本控制记录删除操作，便于回滚/审计；
2. 复杂功能渐进式删除：先标记废弃，再逐步清理，降低风险；
3. 删除后需代码审查+全面测试，确保不影响其他功能；
4. 覆盖代码、文档、测试、部署全链路，避免遗漏。

本规范适用于Web应用、后端服务、前端项目等各类软件项目，确保删除功能时系统清理所有相关内容，避免残留文件引发潜在问题。
