# 代码编码规范（强制）
## 1. 通用命名规范
- 包名：小写，简短无下划线（如 user、goods，禁止 user_service）；
- 变量/函数名：驼峰式（如 userName、GetUserList），禁止下划线；
- 常量名：全大写+下划线（如 MAX_PAGE_SIZE）；
- 结构体名：大驼峰（如 UserRequest、GoodsResponse）。

## 2. Golang开发专属规则
- 变量或函数命名：要用中文定义语义（如 用户名 对应 userName），需要公开的用A_开头（如 A_GetUserList）；
- 禁止生成需要复杂英文配置的代码。

## 3. 注释规范
- 函数/结构体必须加注释，格式：// 函数名 功能描述，接口函数需加 Swagger 注释；
- 注释使用中文，简洁明了，避免无意义注释（如 // 定义变量）。

## 4. 格式规范
- 缩进：4个空格（禁止tab）；
- 导入包排序：标准库 → 第三方库 → 项目内部包，空行分隔；
- 每行代码不超过120个字符，过长需换行。

## 5. 错误处理
- 禁止直接 panic，必须返回 error 并记录日志；
- 错误信息包含上下文（如 fmt.Errorf("查询用户失败：%w", err)）；
- 所有error错误信息使用中文描述。
